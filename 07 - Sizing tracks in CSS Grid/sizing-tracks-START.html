When you use percentages to size your grid the tracks/gaps aren't included. So 'grid-template-columns: 50% 50%;' with 'grid-gap: 20px;' will actually be 100% plus 20px.

fr units(fractional) represent the amount of space left AFTER all the elements are laid out. So if you have one column set to 1fr it will take up whatever space is left or not accounted for. If you do two columns with an 1fr they will split the amount left over. They aren't pixes, just porportions to how much free space is left.

If we do 'grid-template-columns: 200px 2fr 1fr;' then the second column will take up twice as much as the 1fr.

When it comes to rows this will not work by default. The default height of the element is whatever the height of it's child elements are. Whereas the default width is the width of the browser window. So if you want to use 'grid-template-rows: 1fr 1fr 1fr 1fr;' you would need to set the height of the element.

When you use the 'auto' with 'fr' it will make the 'auto' column the size of the content that is in it's elements. Whichever is the largest content in that column will set the width.
